2. Why — The Problem We're Solving

Problem: Customers have no visibility into firmware versions on their servers. They don't know when updates exist until something breaks.

Solution: A Houston module that lists all devices, shows current firmware, notifies of available updates, and allows safe updates from the UI.

Key Rules

  • Risky devices (BIOS, BMC) → display only
  • Safe devices (HBAs, NICs, HDDs, NVMe) → allow update from UI

Scope: Permanent devices — HBAs, NICs, RAID, BIOS, BMC, Seagate HDDs, GPUs

Business Impact

📉
Support Reduction

Fewer tickets about firmware mismatches. Customers self-serve.

😊
Customer Happiness

2-3 hours of manual work becomes 5 minutes.

🛡️
Safety

🔄

3. How — What We're Building

A "Firmware" section in the 45drives-system Cockpit module.

What users see: A table of all components with "Current Version" | "Latest Version" | "Status". Up-to-date → checkmark. Updates available → button. Sidebar badge notifies without opening the page.

4. Architecture

Manifest-driven, cache-first architecture with phased flash enablement.

1. Discovery Script

firmware-discover

  • Enumerates hardware via storcli, mlxup, dmidecode, smartctl, nvme-cli
  • Leverages existing pci script
  • Outputs structured JSON

2. Firmware Repo

manifest.json + binaries

  • Hosted at repo.45drives.com/firmware/
  • 8 component categories
  • Per-entry: model_match, vendor, latest_firmware, flash_tool, sha256

3. Version Check + Cache

firmware-check

  • GPG-verified manifest fetch
  • Compares versions, writes atomic cache
  • Systemd timer runs daily

4. Flash Runner

firmware-flash

  • Downloads binary with SHA256 verification
  • Executes flash command
  • Returns JSON with reboot_needed

5. Sidebar Badge

Preloads on Cockpit start

  • 🔴 Reboot required
  • 🟡 X updates available
  • ✅ Everything current

6. Firmware Table UI

Vue.js component

  • FirmwareTable.vue from cache
  • Status badges, refresh, update buttons
  • Manual instructions for BIOS/BMC

7. HDD Firmware

Seagate Exos via SeaChest

  • Exos X18, X20, X22, X24, 2X18
  • Uses /dev/sgX paths
  • Live microcode (no reboot)

8. Security Pipeline

GPG + SHA256

  • GPG-signed manifest (enforced)
  • SHA256 hash per binary (enforced)
  • Tampered manifests rejected

Open Questions

  • Should HDD flash require unmounting? → Seagate says hot-flash supported
  • Which repo is authoritative? → Thread 9
  • Approval workflow needed? → Phase 1: explicit user confirmation only
